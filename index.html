Explore Pair Programming

What is Pair Programming?

Pair Programming is an agile software development technique in which two programmers work together.  One is the Driver,who writes code while other the Navigator who reviews each line of the code as it is typed in. 

Is it good for me?

Developers learn from each other while coming up with programs and applications with better code quality and fewer bugs.

Why Pair Programming?

Key Advantages from traditional Development Ways:
* Two Heads are better than One - If the driver encounters a hitch with the code, there will be two of them who’ll solve the problem.
* Fewer coding mistakes- Studies shows that it results in 15% fewer bugs than code written by solo programmers.
* An effective way to share knowledge - It can also facilitate mentoring relationships between two programmers.
